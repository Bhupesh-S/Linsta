## ‚ö° QUICK REFERENCE

### API Endpoints by Phase

**Phase 5: Notifications (4 endpoints)**
- `GET /api/notifications` - Get all
- `GET /api/notifications/unread/count` - Count unread
- `PUT /api/notifications/:id/read` - Mark as read
- `PUT /api/notifications/mark-all/read` - Mark all read

**Phase 4: Posts (9 endpoints)**
- `POST /api/posts` - Create
- `GET /api/posts` - Get feed
- `GET /api/posts/:id` - Get one
- `DELETE /api/posts/:id` - Delete
- `POST /api/posts/:id/like` - Like
- `DELETE /api/posts/:id/like` - Unlike
- `POST /api/posts/:id/comment` - Add comment
- `GET /api/posts/:id/comments` - Get comments
- `DELETE /api/posts/:postId/comments/:commentId` - Delete comment

**Phase 3: Events (6 endpoints)**
- `POST /api/events` - Create
- `GET /api/events` - List
- `GET /api/events/:id` - Get one
- `GET /api/events/:id/attendees` - Attendees
- `POST /api/events/:id/rsvp` - RSVP
- `DELETE /api/events/:id/rsvp` - Cancel

**Phase 2: Auth (3 endpoints)**
- `POST /api/auth/register` - Register
- `POST /api/auth/login` - Login
- `POST /api/auth/google` - Google login

**Phase 1: Users (1+ endpoints)**
- `GET /api/users` - List users
- `GET /api/users/:id` - Get profile
- `GET /api/users/profile` - Get own profile
